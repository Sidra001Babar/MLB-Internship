# Text Preprocessing

## What is Text Preprocessing?

Text preprocessing is the process of cleaning and transforming raw text into a structured format that can be understood by machine learning algorithms. Raw text often contains unnecessary information such as punctuation, URLs, numbers, and common words (stopwords) that may not contribute to the model's performance.

The main objectives of text preprocessing are:

- Improve data quality.
- Reduce noise in the text.
- Standardize text into a consistent format.
- Reduce vocabulary size.
- Prepare text for feature extraction techniques such as TF-IDF or Bag of Words.

---

# Text Preprocessing Steps

## 1. Convert Text to Lowercase

### Purpose
Converts all uppercase letters into lowercase letters so that words with different cases are treated as the same word.

Without lowercase conversion, the model considers these as different words:

- Apple
- apple
- APPLE

After conversion, they all become:

- apple

### Why is it important?

- Reduces duplicate words.
- Decreases vocabulary size.
- Improves consistency.

---

## 2. Remove URLs

### Purpose

URLs generally do not provide useful information for most text classification or sentiment analysis tasks. Removing them reduces unnecessary noise in the dataset.

Example:

Before:
```
Visit https://example.com for details.
```

After:
```
Visit for details.
```

### Benefits

- Removes irrelevant information.
- Prevents website links from becoming unnecessary features.

---

## 3. Remove Punctuation

### Purpose

Punctuation marks usually do not carry useful semantic meaning for machine learning models.

Examples include:

- .
- ,
- !
- ?
- :
- ;
- '
- "

Example:

Before:
```
Amazing!!!
```

After:
```
Amazing
```

### Benefits

- Makes text cleaner.
- Prevents punctuation from being treated as separate tokens.

---

## 4. Remove Numbers

### Purpose

Numbers are often removed because they usually do not contribute to understanding the meaning of text in many Natural Language Processing (NLP) tasks.

Example:

Before:
```
I watched it 5 times.
```

After:
```
I watched it times.
```

### Benefits

- Removes unnecessary information.
- Reduces vocabulary size.

> Note: In some applications (such as financial analysis or medical data), numbers may be important and should not be removed.

---

## 5. Tokenization

### What is Tokenization?

Tokenization is the process of breaking a sentence into smaller units called **tokens**.

A token is usually a single word.

Example:

Sentence:

```
This movie is fantastic.
```

Tokens:

- This
- movie
- is
- fantastic

### Why is tokenization important?

Machine learning models cannot process complete sentences directly. They first need the text to be divided into individual words.

---

## 6. Stopword Removal

### What are Stopwords?

Stopwords are very common words that appear frequently in a language but usually carry little useful meaning for machine learning tasks.

Examples:

- the
- is
- are
- am
- was
- were
- and
- of
- in
- to

Example:

Before:

```
This movie is very good
```

After removing stopwords:

```
movie good
```

### Benefits

- Reduces dataset size.
- Removes less informative words.
- Helps focus on important keywords.

---

## 7. Lemmatization

### What is Lemmatization?

Lemmatization converts a word into its **base dictionary form**, called the **lemma**.

It uses vocabulary and linguistic rules to produce actual words.

Examples:

| Original Word | Lemma |
|--------------|-------|
| cars | car |
| studies | study |
| running | running* |
| better | good |
| children | child |

> *By default, "running" remains "running" because the lemmatizer assumes it is a noun unless told otherwise.

---

## Why does "running" not become "run"?

The WordNet Lemmatizer assumes every word is a **noun** by default.

Default behavior:

```
lemmatize("running")
```

Result:

```
running
```

If the word is specified as a verb:

```
lemmatize("running", pos="v")
```

Result:

```
run
```

---

## Meaning of the `pos` Parameter

The `pos` parameter tells the lemmatizer what **part of speech** the word belongs to.

Common values are:

| Parameter | Meaning |
|-----------|---------|
| `pos="n"` | Noun (default) |
| `pos="v"` | Verb |
| `pos="a"` | Adjective |
| `pos="r"` | Adverb |
| `pos="s"` | Adjective Satellite |

Examples:

Without specifying the part of speech:

```
running → running
```

With `pos="v"`:

```
running → run
```

Without specifying:

```
better → better
```

With `pos="a"`:

```
better → good
```

The more accurate the part of speech, the better the lemmatization results.

---

## Lemmatization vs Stemming

Although both techniques reduce words to a simpler form, they work differently.

| Lemmatization | Stemming |
|---------------|----------|
| Uses vocabulary and grammar rules. | Uses simple rule-based word chopping. |
| Produces meaningful dictionary words. | May produce incomplete or incorrect words. |
| Slower because it analyzes word meaning. | Faster because it simply removes prefixes or suffixes. |
| More accurate. | Less accurate. |

### Examples

| Original Word | Lemmatization | Stemming |
|--------------|---------------|----------|
| studies | study | studi |
| running | run* | run |
| caring | care | care |
| better | good | better |
| flies | fly | fli |

(*when the correct part of speech is provided.)

### Which one is better?

- **Stemming** is faster and suitable when speed is more important than linguistic accuracy.
- **Lemmatization** is generally preferred for most NLP and machine learning tasks because it produces meaningful words.

---

## 8. Joining Tokens

### Purpose

After cleaning, tokenizing, removing stopwords, and lemmatizing, the words are joined back together into a single sentence.

Example:

Tokens:

- movie
- absolutely
- amazing

After joining:

```
movie absolutely amazing
```

### Why is this step necessary?

Many feature extraction techniques, such as **TF-IDF** and **Bag of Words**, expect each document to be a complete text string rather than a list of individual words.

Joining the tokens prepares the cleaned text for the next stage of the NLP pipeline.

# Feature Extraction

## What is Feature Extraction?

Machine learning algorithms cannot understand text directly. They only work with numerical data. Therefore, after preprocessing, the cleaned text must be converted into numbers.

This conversion process is called **Feature Extraction**.

A feature represents an important piece of information extracted from the text. Feature extraction transforms each document into a numerical vector while preserving as much useful information as possible.

Example:

Cleaned Text:
```
movie absolutely amazing
```

After Feature Extraction:

```
[0.12, 0.45, 0.00, 0.78, ...]
```

Each number represents the importance of a particular word in the document.

---

# Common Feature Extraction Techniques

There are several methods for converting text into numerical features.

## 1. Bag of Words (BoW)

Counts how many times each word appears in a document.

Example:

Document:
```
cat dog cat
```

Vocabulary:

```
cat
dog
bird
```

Feature Vector:

```
[2, 1, 0]
```

### Advantages

- Simple to understand.
- Fast to compute.

### Disadvantages

- Ignores word importance.
- Common words receive large counts.
- Does not consider context.

---

## 2. TF-IDF (Term Frequency-Inverse Document Frequency)

TF-IDF improves upon Bag of Words by assigning importance to words.

A word receives:

- High weight if it appears frequently in one document.
- Low weight if it appears in many documents.

This makes common words less important and unique words more important.

Example:

Document A:
```
movie amazing story
```

Document B:
```
movie good acting
```

The word **movie** appears in both documents, so its TF-IDF score is lower.

The word **story** appears only once, so it receives a higher TF-IDF score.

---

## Why use TF-IDF?

Compared to Bag of Words, TF-IDF:

- Reduces the importance of very common words.
- Highlights unique and informative words.
- Often improves text classification performance.

---

# Output of Feature Extraction

After TF-IDF, the text dataset becomes a numerical matrix.

Input:

```
movie amazing acting
```

Output:

```
0.42  0.00  0.58  0.31 ...
```

Each row represents one document.

Each column represents one feature (word or phrase).

This numerical matrix is then used for training machine learning models.


# Numerical Data Preprocessing Pipeline

## 1. Load the Dataset

Read the dataset into a DataFrame and inspect its structure.

**Purpose:**
- Understand the data.
- Identify numerical columns.
- Check data types.

---

## 2. Handle Missing Values

Missing values (NaN) should be handled before training a model.

Common techniques:

- Mean Imputation
- Median Imputation
- Mode Imputation
- Forward Fill
- Backward Fill
- KNN Imputation

**Example**

| Age |
|-----:|
| 20 |
| 25 |
| NaN |
| 30 |

After mean imputation:

| Age |
|-----:|
| 20 |
| 25 |
| 25 |
| 30 |

---

## 3. Handle Duplicate Records

Duplicate rows can bias the model.

**Purpose**

- Remove repeated observations.
- Improve data quality.

---

## 4. Detect and Handle Outliers

Outliers are unusually large or small values.

Example:

```
10
12
15
13
14
500
```

Here, **500** is an outlier.

Common methods:

- IQR (Interquartile Range)
- Z-Score
- Winsorization
- Clipping
- Removing outliers

---

## 5. Feature Scaling

Numerical features often have different ranges.

Example:

| Feature | Range |
|---------|------:|
| Age | 18–60 |
| Salary | 20,000–500,000 |

Without scaling, Salary dominates Age.

Common scaling methods:

### Standardization (Z-score Scaling)

Transforms data to have:
- Mean = 0
- Standard Deviation = 1

Best for:
- Logistic Regression
- SVM
- PCA
- Neural Networks

---

### Normalization (Min-Max Scaling)

Scales values between:

```
0 and 1
```

Best for:
- KNN
- Neural Networks
- Deep Learning

---

### Robust Scaling

Uses:

- Median
- Interquartile Range (IQR)

Best when data contains many outliers.

---

## 6. Transform Skewed Data (Optional)

Some numerical features have highly skewed distributions.

Common transformations:

- Log Transformation
- Square Root Transformation
- Box-Cox Transformation
- Yeo-Johnson Transformation

These help make the data more normally distributed.

---

## 7. Feature Engineering (Optional)

Create new numerical features from existing ones.

Examples:

- Age from Date of Birth
- BMI from Height and Weight
- Total Price = Quantity × Unit Price

---

## 8. Feature Selection (Optional)

Keep only the most useful numerical features.

Common methods:

- Correlation Analysis
- Variance Threshold
- Recursive Feature Elimination (RFE)
- SelectKBest

---

## 9. Split the Dataset

Split the data into:

- Training Set
- Testing Set

This allows the model to learn from one portion of the data and evaluate its performance on unseen data.

---

## 10. Train the Machine Learning Model

Use the preprocessed numerical features to train a machine learning algorithm.
