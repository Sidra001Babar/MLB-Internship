# Cross Validation

## What is Cross Validation?

Cross Validation is a model evaluation technique used to measure how well a machine learning model is likely to perform on **unseen data**. Instead of evaluating the model using a single train-test split, the dataset is divided into multiple parts, and the model is trained and tested several times.

It provides a more reliable estimate of the model's performance and helps detect **overfitting** and **underfitting**.

---

# Why Use Cross Validation?

- Provides a more reliable evaluation of the model.
- Makes better use of the available data.
- Reduces the bias caused by a single train-test split.
- Helps compare different machine learning models.
- Helps detect overfitting and underfitting.

---

# Cross Validation Techniques

## 1. Hold-Out Validation

The dataset is split into two parts:

- Training Set
- Testing Set

Example:

- 80% Training
- 20% Testing

### Advantages

- Simple and fast.
- Suitable for large datasets.

### Disadvantages

- Performance depends on a single split.
- May not represent the entire dataset.

---
```text
Dataset
┌──────────────────────────────────────────────┐
│ Train │ Train │ Train │ Train │ Test │
└──────────────────────────────────────────────┘
```

The model is trained **once** and tested **once**.

---
## 2. K-Fold Cross Validation

The dataset is divided into **K equal-sized folds**.

Process:

1. Select one fold as the testing set.
2. Use the remaining folds for training.
3. Repeat until every fold has been used once as the testing set.
4. Compute the average performance.

Example:

For **5-Fold Cross Validation**

```
Fold 1 → Test
Fold 2-5 → Train

Fold 2 → Test
Fold 1,3,4,5 → Train

...

Fold 5 → Test
Fold 1-4 → Train
```
Example: **5-Fold Cross Validation**

### Iteration 1

```text
[Test] [Train] [Train] [Train] [Train]
```

### Iteration 2

```text
[Train] [Test] [Train] [Train] [Train]
```

### Iteration 3

```text
[Train] [Train] [Test] [Train] [Train]
```

### Iteration 4

```text
[Train] [Train] [Train] [Test] [Train]
```

### Iteration 5

```text
[Train] [Train] [Train] [Train] [Test]
```

Each fold becomes the testing set exactly once.

---

### Advantages

- More reliable than a single train-test split.
- Uses all data for both training and testing.

### Disadvantages

- Takes longer to train.

---

## 3. Stratified K-Fold Cross Validation

Similar to K-Fold, but each fold maintains the **same class distribution** as the original dataset.

Example:

Original dataset:

- 70% Positive
- 30% Negative

Each fold will also contain approximately:

- 70% Positive
- 30% Negative

Each fold keeps the same ratio.

```text
Fold 1 : 70% Positive | 30% Negative
Fold 2 : 70% Positive | 30% Negative
Fold 3 : 70% Positive | 30% Negative
Fold 4 : 70% Positive | 30% Negative
Fold 5 : 70% Positive | 30% Negative
```

Training and testing occur exactly like K-Fold, but every fold preserves the original class distribution.

---
### Advantages

- Best for classification problems.
- Prevents class imbalance in each fold.

---

## 4. Leave-One-Out Cross Validation (LOOCV)

Each observation is used once as the testing set, while all remaining observations are used for training.

If there are **100 samples**, the model is trained **100 times**.

### Advantages

- Uses almost all data for training.
- Very accurate evaluation.

### Disadvantages

- Computationally expensive.
- Slow for large datasets.

---

## 5. Repeated K-Fold Cross Validation

K-Fold Cross Validation is repeated multiple times with different random splits.

Example:

- 5-Fold × 3 Repeats
- Total training rounds = 15

### Advantages

- More stable performance estimate.
- Reduces randomness.

### Disadvantages

- More computationally expensive.

---

## Comparison of Cross Validation Techniques

| Technique | Description | Best For | Advantages | Disadvantages |
|-----------|-------------|----------|------------|---------------|
| Hold-Out Validation | One train-test split | Large datasets | Fast and simple | Performance depends on one split |
| K-Fold Cross Validation | Divides data into K folds | General-purpose ML | Reliable evaluation | More computation |
| Stratified K-Fold | Preserves class distribution in each fold | Classification | Handles imbalanced classes | Slightly slower than K-Fold |
| Leave-One-Out (LOOCV) | Uses one sample for testing each time | Small datasets | Maximum use of data | Very slow for large datasets |
| Repeated K-Fold | Repeats K-Fold multiple times | More reliable evaluation | Stable performance estimate | Highest computational cost |

# Detailed Explantion and Comparison of Stratified 5-Fold Cross Validation with K-Fold

## Stratified 5-Fold

Instead of splitting the records sequentially, Stratified K-Fold distributes the classes so that **each fold has approximately the same class distribution as the original dataset**.

Original Dataset:

- Positive = **6 (60%)**
- Negative = **4 (40%)**

A possible stratified split is:

| Fold | Records | Positive | Negative |
|------|---------|:--------:|:--------:|
| Fold 1 | R1(+), R7(-) | 1 | 1 |
| Fold 2 | R2(+), R8(-) | 1 | 1 |
| Fold 3 | R3(+), R9(-) | 1 | 1 |
| Fold 4 | R4(+), R10(-) | 1 | 1 |
| Fold 5 | R5(+), R6(+) | 2 | 0 |

Notice that the algorithm **tries** to keep the same class distribution in every fold.

Since there are **6 Positive** and **4 Negative** samples, it is impossible to have exactly **1 Positive and 1 Negative** in every fold (that would require 5 Negative samples). Therefore, one fold ends up with two Positive samples.

# Why is Stratified K-Fold Better?

Suppose you have a dataset with:

- **95 Positive**
- **5 Negative**

If you use normal K-Fold, one test fold might contain:

```
20 Positive
0 Negative
```

or another might contain:

```
15 Positive
5 Negative
```

This gives an unfair evaluation because the test sets are very different.

With **Stratified K-Fold**, each fold tries to preserve the original ratio:

```
Original Dataset:
95% Positive
5% Negative

↓

Each Fold:
≈95% Positive
≈5% Negative
```

This makes every train/test split more representative of the original dataset and usually produces a more reliable estimate of model performance.
