# How to Choose the Right Evaluation Metrics for Machine Learning Models

Choosing the right evaluation metric depends on:

- The type of machine learning problem.
- The business objective.
- The importance of different types of errors (False Positives and False Negatives).
- Whether the dataset is balanced or imbalanced.

---

# Step 1: Identify the Type of Machine Learning Problem

| Problem Type | Common Evaluation Metrics |
|--------------|--------------------------|
| Classification | Accuracy, Precision, Recall, F1-Score, ROC-AUC |
| Regression | MAE, MSE, RMSE, R² Score |
| Clustering | Silhouette Score, Davies-Bouldin Index |
| Recommendation Systems | Precision@K, Recall@K, MAP, NDCG |

---

# Step 2: Consider the Dataset Balance

## Balanced Dataset

If all classes have nearly equal numbers of samples, **Accuracy** is usually a good metric.

Example:

| Positive | Negative |
|----------|----------|
| 500 | 500 |

Recommended Metric:

- Accuracy

---

## Imbalanced Dataset

If one class is much larger than the other, accuracy can be misleading.

Example:

| Positive | Negative |
|----------|----------|
| 990 | 10 |

A model predicting every sample as "Positive" achieves:

- Accuracy = **99%**

However, it completely fails to identify the Negative class.

Recommended Metrics:

- Precision
- Recall
- F1-Score
- ROC-AUC

---

# Step 3: Consider the Cost of Errors

## When False Positives Are More Costly

Choose **Precision**.

Precision answers:

> "Of all predicted positive cases, how many are actually positive?"

### Examples

- Spam Detection
- Fraud Alerts
- Email Filtering

Reason:

A false alarm is expensive or annoying.

---

## When False Negatives Are More Costly

Choose **Recall**.

Recall answers:

> "Of all actual positive cases, how many did the model detect?"

### Examples

- Cancer Detection
- Disease Diagnosis
- Intrusion Detection
- Fire Detection

Reason:

Missing a positive case can have serious consequences.

---

## When Both Precision and Recall Are Important

Choose **F1-Score**.

F1-Score balances Precision and Recall.

### Examples

- Sentiment Analysis
- Text Classification
- Fake News Detection
- Customer Complaint Detection

---

## When Overall Correct Predictions Matter

Choose **Accuracy**.

### Examples

- Iris Flower Classification
- Digit Recognition (balanced dataset)
- Student Pass/Fail Prediction (balanced classes)

---

# Step 4: Regression Problems

Regression predicts continuous values.

| Metric | When to Use |
|---------|-------------|
| MAE | When average error is important and outliers should have less influence. |
| MSE | When large errors should be penalized more heavily. |
| RMSE | Same as MSE, but easier to interpret because it is in the original units. |
| R² Score | To measure how well the model explains the variation in the data. |

# Hyperparameter Tuning

## What are Hyperparameters?

Hyperparameters are **configuration settings** that are specified **before training** a machine learning model. They control how the learning algorithm works and influence the model's performance.

Unlike model parameters, hyperparameters are **not learned from the training data**. Instead, they are chosen by the user and can be optimized using hyperparameter tuning techniques.

---

# Parameters vs Hyperparameters

| Parameters | Hyperparameters |
|------------|-----------------|
| Learned automatically during training. | Set by the user before training. |
| Determined from the training data. | Control the learning process. |
| Cannot be directly chosen by the user. | Can be manually selected or tuned. |
| Examples: Weights and Biases in Neural Networks, Coefficients in Linear Regression. | Examples: `n_estimators`, `max_depth`, `k`, `learning_rate`, `C`. |

---

# Why are Hyperparameters Important?

The choice of hyperparameters has a significant impact on model performance.

Poor hyperparameter values may lead to:

- Underfitting
- Overfitting
- Lower prediction accuracy
- Longer training time

Well-chosen hyperparameters help the model:

- Learn better patterns.
- Generalize well to unseen data.
- Improve prediction accuracy.
- Reduce overfitting and underfitting.

---

# What is Hyperparameter Tuning?

Hyperparameter tuning is the process of searching for the **best combination of hyperparameter values** that produces the highest model performance.

Instead of manually trying different values, automated search methods evaluate multiple combinations and select the best one.

---

# Example

Suppose you are training a Decision Tree.

Possible values:

| Hyperparameter | Values |
|---------------|--------|
| max_depth | 3, 5, 7 |
| criterion | gini, entropy |

Possible combinations:

| Combination | max_depth | criterion |
|-------------|----------:|-----------|
| 1 | 3 | gini |
| 2 | 3 | entropy |
| 3 | 5 | gini |
| 4 | 5 | entropy |
| 5 | 7 | gini |
| 6 | 7 | entropy |

Each combination is trained and evaluated.

The combination with the best performance is selected.

---

# Why is Hyperparameter Tuning Important?

Hyperparameter tuning helps to:

- Improve model accuracy.
- Reduce overfitting.
- Reduce underfitting.
- Find the best model configuration automatically.
- Improve model generalization.
- Avoid manually testing many combinations.

---

# GridSearchCV

## What is GridSearchCV?

GridSearchCV is a hyperparameter tuning technique that **tries every possible combination** of the specified hyperparameter values.

It uses **Cross Validation** to evaluate each combination and selects the one with the best average performance.

---

## How GridSearchCV Works

Suppose we have:

```
max_depth = [3, 5]

criterion = ["gini", "entropy"]
```

GridSearchCV tests:

| Combination | max_depth | criterion |
|-------------|----------:|-----------|
| 1 | 3 | gini |
| 2 | 3 | entropy |
| 3 | 5 | gini |
| 4 | 5 | entropy |

If 5-fold cross validation is used:

- Combination 1 → trained 5 times
- Combination 2 → trained 5 times
- Combination 3 → trained 5 times
- Combination 4 → trained 5 times

Total model trainings:

```
4 × 5 = 20
```

Finally, GridSearchCV selects the combination with the highest average cross-validation score.

---

## Advantages of GridSearchCV

- Finds the optimal combination from the given search space.
- Easy to use.
- Produces reliable results.
- Uses cross validation for robust evaluation.

---

## Disadvantages of GridSearchCV

- Computationally expensive.
- Slow when many hyperparameters or values are provided.
- Tests combinations that may not improve performance.

---

# RandomizedSearchCV

## What is RandomizedSearchCV?

RandomizedSearchCV is a hyperparameter tuning technique that **randomly selects a specified number of hyperparameter combinations** instead of evaluating every possible combination.

---

## How RandomizedSearchCV Works

Suppose we have:

```
max_depth = [3,5,7,9]

criterion = ["gini","entropy"]

min_samples_split = [2,4,6]
```

Total possible combinations:

```
4 × 2 × 3 = 24
```

Instead of testing all 24 combinations, RandomizedSearchCV may randomly choose only 8 combinations.

Example:

| Combination | max_depth | criterion | min_samples_split |
|-------------|----------:|-----------|------------------:|
| 1 | 3 | gini | 2 |
| 2 | 9 | entropy | 6 |
| 3 | 5 | entropy | 4 |
| 4 | 7 | gini | 2 |
| ... | ... | ... | ... |

Only the selected combinations are evaluated.

---

## Advantages of RandomizedSearchCV

- Faster than GridSearchCV.
- Suitable for large datasets.
- Efficient when many hyperparameters exist.
- Can find near-optimal solutions with less computation.

---

## Disadvantages of RandomizedSearchCV

- Does not guarantee finding the absolute best combination.
- Results depend on the random search.

---

# GridSearchCV vs RandomizedSearchCV

| Feature | GridSearchCV | RandomizedSearchCV |
|---------|--------------|--------------------|
| Search Method | Tests every possible combination | Tests random combinations |
| Speed | Slower | Faster |
| Computational Cost | High | Lower |
| Best Parameter Guarantee | Yes (within the search space) | No |
| Suitable For | Small to medium search spaces | Large search spaces |

---

# Selecting the Best Model Parameters

After tuning, the search method returns the best-performing hyperparameter values based on the chosen evaluation metric (e.g., Accuracy, F1-Score, ROC-AUC).

### Example

Suppose GridSearchCV evaluates the following combinations:

| max_depth | criterion | Accuracy |
|-----------:|-----------|---------:|
| 3 | gini | 91% |
| 3 | entropy | 92% |
| 5 | gini | 95% |
| 5 | entropy | 94% |

The best combination is:

```
max_depth = 5
criterion = "gini"
Accuracy = 95%
```

These become the **best model parameters**, and the final model is trained using them.

---
