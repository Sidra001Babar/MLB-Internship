# Python File Open

## Introduction

**File handling** is the process of creating, reading, updating, and deleting files.

It is an essential part of many Python applications because data is often stored in files instead of being kept only in memory.

Examples of files include:

- Text files (`.txt`)
- CSV files (`.csv`)
- JSON files (`.json`)
- Images (`.jpg`, `.png`)
- PDF files (`.pdf`)

Python provides the built-in **`open()`** function to work with files.

---

# The `open()` Function

The `open()` function is used to open a file.

### Syntax

```python
open(filename, mode)
```

### Parameters

| Parameter | Description |
|-----------|-------------|
| `filename` | The name or path of the file you want to open. |
| `mode` | Specifies how the file should be opened (read, write, append, etc.). |

Example

```python
f = open("demofile.txt")
```

---

# How `open()` Works

When you open a file, Python creates a **file object**.

This object allows you to:

- Read data
- Write data
- Append new data
- Close the file

Example

```python
f = open("demofile.txt")

print(f)
```

Output (may vary)

```python
<_io.TextIOWrapper name='demofile.txt' mode='r' encoding='UTF-8'>
```

---

# File Opening Modes

The second argument of `open()` specifies **how the file should be opened**.

There are four main file modes.

| Mode | Name | Description |
|------|------|-------------|
| `"r"` | Read | Opens a file for reading. Gives an error if the file does not exist. |
| `"a"` | Append | Opens a file for appending new data. Creates the file if it does not exist. |
| `"w"` | Write | Opens a file for writing. Creates the file if it does not exist. If the file exists, its contents are overwritten. |
| `"x"` | Create | Creates a new file. Raises an error if the file already exists. |

---

# 1. Read Mode (`"r"`)

Read mode is the **default mode**.

It opens a file only for reading.

Example

```python
f = open("demofile.txt", "r")
```

If the file **does not exist**, Python raises an error.

Example

```python
f = open("missing.txt", "r")
```

Output

```python
FileNotFoundError
```

---

# 2. Append Mode (`"a"`)

Append mode adds new content **to the end** of a file.

If the file does not exist, Python creates it.

Example

```python
f = open("demofile.txt", "a")
```

Suppose the file contains:

```text
Hello
```

After appending:

```python
f.write(" World")
```

The file becomes:

```text
Hello World
```

> **Note:** Existing content is **not removed**.

---

# 3. Write Mode (`"w"`)

Write mode is used to write data to a file.

If the file exists, **all previous content is erased** before writing new data.

If the file does not exist, Python creates it.

Example

```python
f = open("demofile.txt", "w")
```

Suppose the file originally contains:

```text
Hello World
```

After:

```python
f.write("Python")
```

The file now contains:

```text
Python
```

> **Warning:** `"w"` overwrites the existing file contents.

---

# 4. Create Mode (`"x"`)

Create mode is used to create a **new file only**.

Example

```python
f = open("newfile.txt", "x")
```

If `newfile.txt` already exists, Python raises an error.

Output

```python
FileExistsError
```

This mode helps prevent accidentally overwriting an existing file.

---

# Text Mode and Binary Mode

Besides specifying **what you want to do with the file**, you can also specify **how Python should treat the file's contents**.

There are two modes:

| Mode | Name | Used For |
|------|------|----------|
| `"t"` | Text Mode | Text files (`.txt`, `.csv`, `.json`, `.py`) |
| `"b"` | Binary Mode | Images, videos, audio, PDFs, executable files |

---

## Text Mode (`"t"`)

Text mode is the **default mode**.

Example

```python
f = open("demofile.txt", "rt")
```

Since `"t"` is the default, this is exactly the same as:

```python
f = open("demofile.txt", "r")
```

Or simply:

```python
f = open("demofile.txt")
```

All three statements behave the same way.

---

## Binary Mode (`"b"`)

Binary mode is used for non-text files such as:

- Images
- Videos
- Audio files
- PDF files

Example

```python
image = open("photo.jpg", "rb")
```

Here:

- `"r"` means **read**
- `"b"` means **binary**

---

# Why `open("file.txt")` Works

Consider this statement:

```python
f = open("demofile.txt")
```

Python automatically assumes:

```python
f = open("demofile.txt", "rt")
```

because:

- `"r"` (read) is the default file operation.
- `"t"` (text mode) is the default file type.

So these are all equivalent:

```python
f = open("demofile.txt")
```

```python
f = open("demofile.txt", "r")
```

```python
f = open("demofile.txt", "rt")
```

---



# Text vs Binary Modes

| Mode | File Type | Example |
|------|-----------|---------|
| `"t"` | Text | `.txt`, `.csv`, `.json`, `.py` |
| `"b"` | Binary | `.jpg`, `.png`, `.pdf`, `.mp3`, `.mp4` |

Examples

```python
# Read a text file
f = open("notes.txt", "rt")
```

```python
# Read an image
image = open("photo.jpg", "rb")
```

---

# Important Notes

- `open()` is the main function used for file handling in Python.
- It requires at least the **filename**.
- The **default mode** is `"r"` (read).
- The **default file type** is `"t"` (text).
- `"w"` overwrites existing content.
- `"a"` adds data without deleting existing content.
- `"x"` creates a new file and fails if it already exists.
- Use `"b"` when working with binary files such as images or PDFs.
- Trying to open a non-existent file in `"r"` mode raises a `FileNotFoundError`.

# Python Delete File

## Introduction

Python allows you to **delete files and folders** using the built-in **`os`** module.

The `os` module provides functions for interacting with the operating system, such as:

- Creating files and folders
- Renaming files
- Deleting files
- Deleting directories
- Checking if a file exists

Before using these functions, you must import the `os` module.

```python
import os
```

---

# Delete a File

To delete a file, use the **`os.remove()`** function.

### Syntax

```python
os.remove(filename)
```

### Example

Delete the file **`demofile.txt`**.

```python
import os

os.remove("demofile.txt")
```

If the file exists, it is permanently deleted.

---

# What Happens If the File Does Not Exist?

If you try to delete a file that does not exist, Python raises an error.

Example

```python
import os

os.remove("missing.txt")
```

Output

```python
FileNotFoundError
```

This happens because Python cannot find the specified file.

---

# Check if a File Exists Before Deleting

To avoid getting a `FileNotFoundError`, check whether the file exists before deleting it.

Python provides the **`os.path.exists()`** function for this purpose.

### Syntax

```python
os.path.exists(filename)
```

It returns:

- `True` → if the file exists.
- `False` → if the file does not exist.

---

## Example

```python
import os

if os.path.exists("demofile.txt"):
    os.remove("demofile.txt")
else:
    print("The file does not exist")
```

### Output (if the file exists)

```python
# File is deleted
```

### Output (if the file does not exist)

```python
The file does not exist
```

---

# How It Works

```python
if os.path.exists("demofile.txt"):
```

Checks whether the file exists.

If it returns **`True`**:

```python
os.remove("demofile.txt")
```

The file is deleted.

Otherwise:

```python
print("The file does not exist")
```

A message is displayed instead of raising an error.

---

# Delete a Folder

To delete an entire folder (directory), use **`os.rmdir()`**.

### Syntax

```python
os.rmdir(folder_name)
```

### Example

Delete the folder **`myfolder`**.

```python
import os

os.rmdir("myfolder")
```

If the folder is empty, it is deleted successfully.

---

# Important Limitation of `os.rmdir()`

`os.rmdir()` can **only delete empty folders**.

Example

```
myfolder/
```

Since it is empty:

```python
os.rmdir("myfolder")
```

 Folder is deleted.

---

Suppose the folder contains a file:

```
myfolder/
    notes.txt
```

Then:

```python
os.rmdir("myfolder")
```

Output

```python
OSError: Directory not empty
```

Python refuses to delete a folder that still contains files or subfolders.

---

# Difference Between `os.remove()` and `os.rmdir()`

| Function | Purpose | Works On |
|----------|----------|----------|
| `os.remove()` | Deletes a file | Files only |
| `os.rmdir()` | Deletes a folder | Empty folders only |

---

# Python JSON

## What is JSON?

**JSON (JavaScript Object Notation)** is a lightweight format used for **storing** and **exchanging** data.

Although JSON is based on JavaScript object syntax, it is a **language-independent** format that is supported by many programming languages, including Python.

JSON is commonly used for:

- APIs (sending and receiving data)
- Configuration files
- Data storage
- Communication between applications

Example of JSON data:

```json
{
    "name": "John",
    "age": 30,
    "city": "New York"
}
```

---

# JSON in Python

Python provides a built-in module called **`json`** for working with JSON data.

Before using it, import the module.

```python
import json
```

The `json` module allows you to:

- Convert JSON → Python
- Convert Python → JSON
- Format JSON for better readability

---

# Parse JSON (Convert JSON to Python)

If you have JSON data stored as a **string**, you can convert it into a Python object using:

```python
json.loads()
```

The word **loads** means:

> **Load JSON from a string.**

---

## Syntax

```python
json.loads(json_string)
```

---

## Example

```python
import json

# JSON string
x = '{ "name":"John", "age":30, "city":"New York"}'

# Convert JSON to Python dictionary
y = json.loads(x)

print(y["age"])
```

### Output

```python
30
```

---

## How It Works

JSON string:

```json
{
    "name":"John",
    "age":30,
    "city":"New York"
}
```

After:

```python
y = json.loads(x)
```

Python converts it into a **dictionary**.

```python
{
    "name": "John",
    "age": 30,
    "city": "New York"
}
```

Now you can access values like a normal dictionary.

```python
print(y["name"])
```

Output

```python
John
```

---

# Convert Python to JSON

If you have a Python object, convert it into a JSON string using:

```python
json.dumps()
```

The word **dumps** means:

> **Dump Python data into a JSON string.**

---

## Syntax

```python
json.dumps(python_object)
```

---

## Example

```python
import json

x = {
    "name": "John",
    "age": 30,
    "city": "New York"
}

y = json.dumps(x)

print(y)
```

### Output

```json
{"name": "John", "age": 30, "city": "New York"}
```

The output is a **JSON string**.

---

# `loads()` vs `dumps()`

| Method | Converts From | Converts To |
|---------|---------------|-------------|
| `json.loads()` | JSON string | Python object |
| `json.dumps()` | Python object | JSON string |

Remember:

- **loads** → JSON → Python
- **dumps** → Python → JSON

---

# Complex Python Object

JSON can also represent nested objects and arrays.

Example

```python
import json

x = {
    "name": "John",
    "age": 30,
    "married": True,
    "divorced": False,
    "children": ("Ann", "Billy"),
    "pets": None,
    "cars": [
        {
            "model": "BMW 230",
            "mpg": 27.5
        },
        {
            "model": "Ford Edge",
            "mpg": 24.1
        }
    ]
}

print(json.dumps(x))
```

### Output (single line)

```json
{"name":"John","age":30,"married":true,"divorced":false,"children":["Ann","Billy"],"pets":null,"cars":[{"model":"BMW 230","mpg":27.5},{"model":"Ford Edge","mpg":24.1}]}
```

Notice:

- Tuple becomes an array.
- `True` becomes `true`.
- `False` becomes `false`.
- `None` becomes `null`.

---

# Formatting JSON

By default, `json.dumps()` produces JSON on a single line.

For better readability, use the **`indent`** parameter.

---

## Using `indent`

Example

```python
import json

print(json.dumps(x, indent=4))
```

### Output

```json
{
    "name": "John",
    "age": 30,
    "married": true,
    "divorced": false,
    "children": [
        "Ann",
        "Billy"
    ],
    "pets": null
}
```

The `indent=4` argument tells Python to use **4 spaces** for each indentation level.

---

# Using `separators`

The **`separators`** parameter changes how keys and values are separated.

Default separators are:

```python
(", ", ": ")
```

Meaning:

- Comma followed by a space separates items.
- Colon followed by a space separates keys and values.

Example output:

```json
{
    "name": "John",
    "age": 30
}
```

---

## Custom Separators

Example

```python
import json

print(json.dumps(x, indent=4, separators=(". ", " = ")))
```

Output

```text
{
    "name" = "John".
    "age" = 30
}
```

Here:

- `". "` replaces the comma separator.
- `" = "` replaces the colon separator.

This formatting is mainly used for custom display purposes and is **not standard JSON**.

# Why is JSON Commonly Used?

JSON is one of the most popular data formats because it is:

- Easy to read and write.
- Lightweight (uses very little space).
- Human-readable.
- Supported by almost every programming language.
- Ideal for transferring data between servers and applications.
- Commonly used in APIs, web applications, configuration files, and databases.

# Common Uses of JSON

- Web APIs
- Configuration files
- Data storage
- Communication between frontend and backend
- Mobile applications
- Machine learning datasets

For example, when a weather website sends today's weather to your browser, it often sends the data as JSON.

Example:

```json
{
    "city": "Lahore",
    "temperature": 38,
    "condition": "Sunny"
}
```
