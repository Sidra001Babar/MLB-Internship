# Random Imputation

**Definition**

Random Imputation replaces each missing value with a **randomly selected value from the same column**.

### When to Use
- Can be used for both **numerical** and **categorical** features.
- Useful when you want to preserve the original data distribution.

### Advantages
- Preserves the data distribution.
- Maintains the variance better than mean or median imputation.
- Can be applied to both numerical and categorical data.

### Disadvantages
- Random values are generated every time unless a fixed random state is used.
- Extra handling is required in production to ensure consistent results.
- Less effective for Decision Tree models.

> **Note:** In production, ensure that the same missing input always receives the same imputed value.


# Randomly Select Existing Values

Randomly select values from the non-missing data using the `sample()` function.

- `n` = number of missing values
- `random_state` ensures the same random values are selected every time.

```python
random_values = df['GarageQual'].dropna().sample(
    n=len(missing_index),
    random_state=42,
    replace=True
)
```

# Assign Random Values

Replace the missing values with the randomly selected values.

```python
random_values.index = missing_index

df.loc[missing_index, 'GarageQual_random'] = random_values
```
