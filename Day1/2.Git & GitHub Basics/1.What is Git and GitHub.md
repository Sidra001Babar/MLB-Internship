# Difference Between Git and GitHub

| **Feature** | **Git** | **GitHub** |
|--------------|---------|------------|
| **Definition** | Git is a distributed version control system used to track changes in source code. | GitHub is a cloud-based platform that hosts Git repositories and provides collaboration features. |
| **Type** | Software (Version Control System) | Web-based hosting service and collaboration platform |
| **Purpose** | Manages and tracks changes in code locally. | Stores Git repositories online and enables team collaboration. |
| **Internet Requirement** | Does not require an internet connection for most operations. | Requires an internet connection to access remote repositories and collaboration features. |
| **Installation** | Must be installed on your local machine. | Can be accessed through a web browser; Git is needed for local operations. |
| **Repository Storage** | Stores repositories on the local computer. | Stores repositories on remote cloud servers. |
| **Collaboration** | Limited collaboration through manual sharing of repositories. | Provides pull requests, code reviews, issue tracking, and team collaboration tools. |
| **Authentication** | No account is required for local use. | Requires a GitHub account to create and manage remote repositories. |
| **Command Line** | Primarily used through Git commands in the terminal or command prompt. | Can be managed through a web interface, Git commands, or GitHub Desktop. |
| **Branch Management** | Supports creating and merging branches locally. | Supports branch management along with pull requests and branch protection rules. |
| **Open Source** | Completely open-source software. | Offers both free and paid plans with additional enterprise features. |
| **Examples of Commands/Features** | `git init`, `git add`, `git commit`, `git branch`, `git merge` | Pull Requests, Issues, Actions, Projects, Discussions, Releases |
