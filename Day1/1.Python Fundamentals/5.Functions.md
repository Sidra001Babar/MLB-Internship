## 10. Positional-Only and Keyword-Only Arguments in Python

Python allows you to control **how arguments are passed** to a function.

There are three ways to pass arguments:

1. Positional Arguments
2. Keyword Arguments
3. Positional-only and Keyword-only Arguments

---

# 1. Positional-Only Arguments (`/`)

A **positional-only argument** can **only** be passed according to its position.

To make an argument positional-only, place a `/` after the positional arguments.

## Syntax

```python
def function_name(arg1, arg2, /):
    pass
```

Everything **before** `/` is positional-only.

---

## Example 1

```python
def my_function(name, /):
    print("Hello", name)

my_function("Emil")
```

### Output

```
Hello Emil
```

### Explanation

The value `"Emil"` is passed according to its position.

- First argument → `name`
- This is correct.

---

## Incorrect Example

```python
def my_function(name, /):
    print("Hello", name)

my_function(name="Emil")
```

### Output

```
TypeError:
my_function() got some positional-only arguments passed as keyword arguments: 'name'
```

### Why?

Since `name` is before `/`, Python does **not** allow writing

```python
name="Emil"
```

It must be

```python
my_function("Emil")
```

---

## Without `/`

```python
def my_function(name):
    print("Hello", name)

my_function(name="Emil")
```

### Output

```
Hello Emil
```

### Explanation

Without `/`, Python allows **both**

```python
my_function("Emil")
```

and

```python
my_function(name="Emil")
```

Both are valid.

---

# Why Use Positional-Only Arguments?

Suppose a function calculates a square.

```python
def square(number, /):
    return number * number
```

Correct

```python
square(5)
```

Incorrect

```python
square(number=5)
```

This keeps the function simple and prevents users from relying on parameter names.

---

# 2. Keyword-Only Arguments (`*`)

A **keyword-only argument** must be passed using its parameter name.

Place a `*` before the keyword-only arguments.

## Syntax

```python
def function_name(*, arg1):
    pass
```

Everything **after** `*` becomes keyword-only.

---

## Example

```python
def my_function(*, name):
    print("Hello", name)

my_function(name="Emil")
```

### Output

```
Hello Emil
```

### Explanation

Here,

```python
name="Emil"
```

must be written.

---

## Incorrect Example

```python
def my_function(*, name):
    print("Hello", name)

my_function("Emil")
```

### Output

```
TypeError:
my_function() takes 0 positional arguments but 1 was given
```

### Why?

Because `name` is keyword-only.

Python expects

```python
name="Emil"
```

not

```python
"Emil"
```

---

## Without `*`

```python
def my_function(name):
    print("Hello", name)

my_function("Emil")
```

Output

```
Hello Emil
```

You can also write

```python
my_function(name="Emil")
```

Both are allowed because there is no `*`.

---

# Why Use Keyword-Only Arguments?

Suppose you have a function that creates a user.

```python
def create_user(*, username, age):
    print(username, age)
```

Correct

```python
create_user(username="Ali", age=20)
```

Incorrect

```python
create_user("Ali", 20)
```

Using keywords makes the code much more readable.

---

# 3. Normal Arguments

If you use neither `/` nor `*`, arguments can be passed either way.

```python
def greet(name):
    print(name)
```

Both work.

```python
greet("Ali")
```

```python
greet(name="Ali")
```

Output

```
Ali
```

---

# 4. Using Both `/` and `*`

You can use both symbols in the same function.

```python
def student(name, roll_no, /, class_name, *, grade):
    print(name)
    print(roll_no)
    print(class_name)
    print(grade)
```

### Function Call

```python
student("Ali", 101, "BSCS", grade="A")
```

Output

```
Ali
101
BSCS
A
```

---

## Incorrect Calls

### Using keyword for positional-only argument

```python
student(name="Ali", 101, "BSCS", grade="A")
```

Error because `name` is positional-only.

---

### Using positional value for keyword-only argument

```python
student("Ali", 101, "BSCS", "A")
```

Error because `grade` is keyword-only.

---

# Visual Summary

| Symbol | Meaning | Example |
|---------|----------|---------|
| `/` | Arguments before it are positional-only | `func(a, /)` |
| `*` | Arguments after it are keyword-only | `func(*, b)` |
| No `/` or `*` | Arguments can be positional or keyword | `func(a)` |

---

# Complete Example

```python
def demo(a, b, /, c, d, *, e, f):
    print(a, b, c, d, e, f)

demo(1, 2, 3, 4, e=5, f=6)
```

Here:

- `a` → positional-only
- `b` → positional-only
- `c` → positional or keyword
- `d` → positional or keyword
- `e` → keyword-only
- `f` → keyword-only

Valid call

```python
demo(1, 2, 3, 4, e=5, f=6)
```

Also valid

```python
demo(1, 2, c=3, d=4, e=5, f=6)
```

Invalid

```python
demo(a=1, b=2, c=3, d=4, e=5, f=6)
```

Reason: `a` and `b` are positional-only.

Invalid

```python
demo(1, 2, 3, 4, 5, 6)
```

Reason: `e` and `f` are keyword-only.
